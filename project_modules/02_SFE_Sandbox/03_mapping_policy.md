# 03 Mapping Policy

## 목적
`config/mapping.json` 운영 기준(추가/변경/하위호환/리뷰/이력)을 정의한다.

## 운영 원칙
- 매핑은 표준 컬럼 체계로 수렴한다.
- 기존 표준키 삭제 금지(하위호환 유지).
- 원천 별칭은 누적 등록하고 충돌은 검토 후 병합한다.

## 변경 유형
- 추가: 신규 원천 별칭 등록
- 수정: 오매핑 교정
- 폐기: 직접 삭제 대신 비활성/대체 경로 기록

## 리뷰 절차
1. 변경 요청 등록
2. 영향 범위 확인(시나리오/도메인)
3. 샘플 데이터 검증
4. 승인 후 반영
5. 변경 로그 기록

## 이력 관리 항목
- 변경일
- 변경자
- 변경 사유
- 영향 도메인/시나리오
- 롤백 방법

## 품질 기준
- 표준 컬럼 커버리지 유지
- 신규 매핑으로 기존 결과물 훼손 금지

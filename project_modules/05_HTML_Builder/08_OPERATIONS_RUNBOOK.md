# Operations Runbook

작성일: 2026-03-12

## 1. 목적

이 문서는 HTML Builder를 `공통 출력 유틸`로 사용할 때의 표준 절차다.

## 2. 기본 절차

1. 어떤 미리보기를 만들지 결정한다.
2. 해당 템플릿을 고른다.
3. 템플릿 계약에 맞는 payload가 준비됐는지 확인한다.
4. HTML을 생성한다.
5. 브라우저에서 빠르게 확인한다.
6. 필요하면 상세 파일 다운로드 참조를 함께 건다.

## 3. 확인할 항목

- 템플릿 ID가 맞는지
- payload 필수 항목이 다 있는지
- 깨진 카드/표/지도 블록이 없는지
- 다운로드 링크가 맞는지

## 4. 장애 대응

### 템플릿이 깨질 때
- 템플릿 파일 경로와 버전을 먼저 확인

### 데이터가 비어 보일 때
- payload 필드 누락 여부 확인
- 앞단 모듈 산출물과 계약이 맞는지 확인

### HTML이 너무 무거울 때
- 상세 테이블을 다운로드 파일로 분리
- 지도/차트 payload를 줄여서 다시 생성

## 5. 운영 원칙

- Builder를 별도 서비스처럼 운영하지 않는다.
- 템플릿과 payload 계약을 같이 본다.
- HTML은 공식 계산 결과가 아니라 최종 표현 단계다.

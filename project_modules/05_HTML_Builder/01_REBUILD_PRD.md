# HTML Output Utility PRD

작성일: 2026-03-12  
문서 버전: v1.0  
대상: `05_HTML_Builder` 문서 재정리

## 1. 한 줄 정의

HTML Builder는 `독립 제품`이 아니라,  
`분석 결과를 사람이 바로 볼 수 있는 HTML 미리보기로 빠르게 바꿔주는 템플릿 기반 출력 자동화 유틸`이다.

## 2. 왜 필요한가

- 대시보드나 지도 결과를 사람이 매번 다시 꾸미면 시간이 오래 걸린다.
- 같은 종류의 보고서를 반복해서 만들 때 형식이 흔들리기 쉽다.
- 결과를 빨리 공유하지 못하면 분석 가치가 늦게 전달된다.

즉 이 모듈의 핵심 가치는 `새 계산`이 아니라 아래 3가지다.

1. 결과 전달 시간 단축
2. 반복 작업 감소
3. 결과 형식 표준화

## 3. 제품 정체성

이 모듈은 다음과 같이 이해한다.

- `Sandbox / TO / PDF / CRM` = 계산 모듈
- `HTML Builder` = 출력 자동화 계층

중요:

- Builder는 raw를 직접 읽지 않는다.
- Builder는 독립 Result Asset을 만들지 않는다.
- Builder는 승인된 결과 또는 템플릿 계약에 맞춘 payload를 받아 HTML을 만든다.

## 4. 현재 구조 해석

현재 Builder 흐름은  
`데이터가 먼저, 템플릿이 나중`이 아니라  
`보여줄 미리보기 목표가 먼저, 템플릿이 먼저`인 구조에 가깝다.

즉 실제 흐름은 아래처럼 이해한다.

```text
미리보기 목표 정의
-> 템플릿 확정
-> 템플릿 계약에 맞춘 데이터 생산
-> Template Injection
-> HTML
```

공용 세계관 안에서는 아래처럼 본다.

```text
원천데이터 -> Adapter -> Module -> Result Asset -> OPS -> Template Contract -> Template Injection -> HTML
```

## 5. 이 모듈이 하는 일

- 모듈 결과를 사람이 보기 쉬운 HTML로 바꾼다.
- 대시보드/지도/검증 미리보기 형식을 맞춘다.
- 템플릿 계약에 맞게 payload를 주입한다.
- 무거운 상세 데이터는 다운로드 참조로 분리한다.

## 6. 이 모듈이 하지 않는 일

- raw 업로드 처리
- 독립 API 서비스 운영
- 인증/인가 중심의 별도 플랫폼 역할
- 비동기 job 시스템 운영
- 분석 로직 계산
- OPS 평가 대체

## 7. 핵심 요구사항

### 7.1 기능 요구
- 승인된 결과 또는 승인된 payload만 입력으로 받는다.
- 템플릿 기준으로 동일 형식 HTML을 빠르게 만든다.
- 지도, 대시보드, 검증표, 요약카드 같은 반복 출력을 표준화한다.

### 7.2 출력 원칙
- 미리보기는 가볍게 유지한다.
- 상세 원본은 다운로드 파일로 분리한다.
- 템플릿이 요구하지 않는 계산을 Builder 안에서 새로 하지 않는다.

### 7.3 운영 원칙
- Builder는 별도 플랫폼으로 키우지 않는다.
- 템플릿 계약과 payload 규격을 명확히 유지한다.
- 새 템플릿을 추가하더라도 기존 모듈 계산 로직을 흔들지 않는다.

## 8. 성공 기준

1. 반복 보고서 제작 시간이 줄어든다.
2. 같은 종류 출력의 형식이 흔들리지 않는다.
3. 대시보드/지도/검증 미리보기를 빠르게 공유할 수 있다.
4. 템플릿 변경이 계산 모듈 변경 없이 가능한 범위가 늘어난다.

## 9. 냉정한 평가

잘한 점:
- 결과 전달 시간을 줄이는 데 직접 기여한다.
- 사람이 다시 손으로 꾸미는 일을 줄여준다.
- 포트폴리오나 데모에서 결과를 빨리 보여주기 좋다.

아쉬운 점:
- 독립 제품처럼 크게 설계하면 역할 대비 과하다.
- API/보안/비동기/관측성을 다 넣으면 본질보다 껍데기가 커진다.
- 계산 모듈처럼 독립 모듈 가치를 주장하기 어렵다.

개선 방향:
- `출력 자동화 유틸`로 위치를 명확히 한다.
- 템플릿 계약과 payload 규격에 집중한다.
- Builder를 크게 만들기보다 공통 렌더 계층으로 얇게 유지한다.

## 10. 비범위

- 문서 업로드 플랫폼
- 멀티테넌트 API 서비스
- 인증/인가/레이트리밋 중심 운영 시스템
- 큐/잡 기반 대규모 서비스 구조
- Builder 안에서 새 계산 엔진 개발

# Template Output Policy

작성일: 2026-03-12

## 1. 목적

이 문서는 Builder 출력 원칙을 짧게 고정한다.

## 2. 출력 원칙

1. 템플릿이 먼저다.
2. 데이터는 템플릿 계약에 맞게 정리한다.
3. 미리보기는 가볍게 유지한다.
4. 무거운 원본은 다운로드로 분리한다.
5. Builder 안에서 새 계산을 하지 않는다.

## 3. 미리보기 유형

- 요약 대시보드
- 지도 미리보기
- 검증 요약표
- 통합 허브형 HTML

## 4. 금지할 것

- 모든 데이터를 HTML 한 장에 다 싣기
- 템플릿 없이 즉석 렌더를 범용 기능처럼 키우기
- 계산 로직을 템플릿 안에 넣기

## 5. 한 줄 원칙

`빠르게 보여주되, 가볍게 보여준다.`

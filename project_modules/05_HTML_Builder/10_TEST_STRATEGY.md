# Test Strategy

작성일: 2026-03-12

## 1. 테스트 목적

Builder 테스트의 목표는 `서비스 운영성`이 아니라  
`템플릿 계약이 깨지지 않고 HTML이 정상 출력되는지`를 확인하는 것이다.

## 2. 필수 테스트

### Contract Test
- 템플릿 계약 필수 필드 존재 확인
- payload와 template_contract 정합성 확인

### Render Smoke
- 대표 템플릿 1개씩 실제 HTML 생성 확인
- 브라우저에서 기본 렌더 깨짐 없는지 확인

### Snapshot Check
- 카드/섹션 구조가 의도치 않게 바뀌지 않았는지 확인

### Download Reference Check
- 미리보기에서 다운로드 참조가 정상 연결되는지 확인

## 3. 무거운 테스트보다 중요한 것

- 새 계산을 하지 않는지
- payload 누락 시 어디가 깨지는지 바로 보이는지
- 템플릿 변경이 다른 미리보기를 망가뜨리지 않는지

## 4. 한 줄 원칙

`Builder 테스트는 API 테스트보다 템플릿 계약 테스트가 더 중요하다.`

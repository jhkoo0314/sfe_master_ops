# Domain Glossary

작성일: 2026-03-12

## 용어 정의

- `Result Asset`
  - 계산 모듈이 만든 공식 결과 요약

- `Template Contract`
  - 어떤 HTML을 만들지 먼저 정한 계약
  - 필요한 카드, 표, 지도, 다운로드 참조를 정의한다

- `Render Payload`
  - 템플릿 계약에 맞게 정리된 실제 입력 데이터

- `Template Injection`
  - payload를 템플릿에 넣어 HTML을 만드는 마지막 단계

- `Preview HTML`
  - 사람이 바로 확인하는 가벼운 결과 화면

- `Template Pack`
  - 템플릿 파일과 관련 설정 묶음

- `Preview Goal`
  - 무엇을 보여줄지에 대한 목적
  - 예: CRM 요약, Territory 지도, Prescription 검증표

- `Download Reference`
  - 미리보기에서 다 담지 않는 상세 파일 참조

- `Output Utility`
  - 계산하지 않고 결과를 빠르게 보여주는 보조 계층

## 금지할 오해

- Builder를 계산 모듈이라고 부르지 않는다.
- Builder 결과를 공식 Result Asset이라고 부르지 않는다.
- 템플릿이 없는 상태에서 범용적으로 다 렌더한다고 가정하지 않는다.

## 🚨 슬라이드 3: Problem Statement

**기존 CRM의 근본적 한계**

**❌ 입력 중심, 분석 부재**
- 영업사원이 보고서 작성에 시간 낭비
- 기록된 데이터는 방문 횟수 수준에 머무름
- 관리자는 '감'에 의존한 의사결정

**❌ 행동의 질적 측정이 불가능**
- "많이 뛰었다"는 양적 평가만 가능
- "어떻게 뛰었는지" 구체적 분석 부재
- 효과적인 행동 패턴 표준화 불가

**❌ 성과의 원인 분석 불가**
- 결과는 나오지만 "왜 그 결과가 나왔는지" 설명 불가
- 우수 사원의 성공 요인 파악 불가

## 🎯 슬라이드 6: Behavior Layer 소개

**Behavior Layer: 행동 측정의 핵심**

**영업 프로세스를 8가지 표준화된 행동으로 분류**

**1. 접근 (Approach)** - 고객과의 첫 만남
**2. 컨택 (Contact)** - 지속적인 소통
**3. 대면 (Visit)** - 직접 방문 상담
**4. 프레젠테이션 (Presentation)** - 제품 소개
**5. 질문/탐색 (Question)** - 니즈 파악
**6. 필요성 자극 (Need Creation)** - 잠재 니즈 발굴
**7. 시연/자료제공 (Demonstration)** - 제품 시연
**8. 후속관리 (Follow-up)** - 방문 후 관리

---

## 📊 슬라이드 7: 행동 측정 방식

**행동의 양적 + 질적 측정**

**양적 측정 (Quantity)**
- 빈도: 기간 내 발생 횟수
- 강도: 지속 시간 및 집중도
- 일관성: 규칙적 실행 정도

**질적 측정 (Quality)**
- 전략성: 목적성 및 계획성
- 효과성: 실제 성과 기여도
- 적응성: 상황별 최적화 정도

**결과**: 각 행동별 품질 점수 (0-100점)

---

## 🔢 슬라이드 8: Behavior Score 계산

**3차원 Behavior Score 계산**

**1. 행동 강도 점수 (Intensity Score)**
```
Intensity = (가중치 합계 / 최대 기대치) × 100

가중치: 방문=3, 전화=2, 메시지=1, 프레젠테이션=2, 후속=1
```

**2. 행동 다양성 점수 (Diversity Score)**
```
Diversity = (실행 행동 종류 수 / 8) × 100
```

**3. 행동 질 점수 (Quality Score)**
```
Quality = (평균 품질 × 0.4) + (평균 양 × 0.3) + (후속 비율 × 30)
```

---

## 📈 슬라이드 9: Analytics Engine

**Analytics Engine: 지능적 분석 코어**

**Pearson 상관계수 기반 분석**
```
Correlation = Σ((Xᵢ - X̄)(Yᵢ - Ȳ)) / √[Σ(Xᵢ - X̄)² Σ(Yᵢ - Ȳ)²]
```

- **X**: 특정 행동의 품질 점수
- **Y**: 성과 지표 (HIR, 전환률 등)
- **결과**: -1 ~ +1 사이의 상관계수

**실시간 상관관계 분석으로 어떤 행동이 성과에 가장 큰 영향을 미치는지 식별**

---

## 🎯 슬라이드 10: Outcome Layer 소개

**Outcome Layer: 성과 측정의 종합**

**4가지 핵심 성과 지표 (KPI)**

**1. 고효과 행동 비율 (HIR)**
- 고품질 행동의 비율 × 평균 영향력
- 효과적인 행동의 실행 정도 측정

**2. 전환률 (Conversion Rate)**
- 행동 → 매출 전환 성공률
- 행동의 실제 성과 기여도 측정

**3. 필드 성장률 (Field Growth Rate)**
- 담당 구역의 성장 정도 측정

**4. 처방 기반 성과지수 (Prescription Index)**
- 종합적인 영업 성과 측정

---

## 🧮 슬라이드 11: HIR 계산 로직

**HIR (High-Impact Rate) 상세 계산**

```
HIR = Σ(행동ᵢ × 품질점수ᵢ × 영향력계수ᵢ) / 총행동수
```

**계산 요소:**
- **행동ᵢ**: 실행 여부 (0 또는 1)
- **품질점수ᵢ**: 0-100점 사이 품질 평가
- **영향력계수ᵢ**: 상관관계 분석 결과 (0.1-2.0)

**HIR 해석:**
- **80-100점**: 탁월한 행동 실행
- **60-79점**: 우수한 행동 실행
- **40-59점**: 보통 수준
- **0-39점**: 개선 필요

## 🤖 슬라이드 12: AI 코칭 시스템 개요

**AI 코칭 시스템: 자동 코칭 플랫폼**

**6가지 코칭 신호 타입 자동 생성**

**🚨 High Priority:**
- 관계 악화 경보 (고객 이탈 리스크)
- 경쟁사 등장 경보 (시장 점유율 위협)
- 전환 행동 부족 경보 (매출 타격)

**⚠️ Medium Priority:**
- 행동 부족 경보 (일반 개선 필요)
- 관심도 급하락 경보 (관계 악화 초기)

**ℹ️ Low Priority:**
- 취약 행동 경보 (장기적 개선)

---

## 🚨 슬라이드 13: 코칭 신호 상세 (1/2)

**코칭 신호 감지 로직**

**1. 행동 부족 경보**
```
기준치 = 기간 ≤ 7일 ? 1회 : 기간 ≤ 30일 ? 3회 : 5회
if (실행횟수 < 기준치) → 경보
```

**2. 관계 악화 경보**
```
감소율 = (현재빈도 - 이전빈도) / 이전빈도
if (감소율 < -0.5) → High Priority 경보
```

**3. 경쟁사 등장 경보**
- Activity 로그에서 경쟁사 키워드 자동 탐지
- "요즘 ○○사에서...", 가격 문의 급증 등

---

## 🚨 슬라이드 14: 코칭 신호 상세 (2/2)

**코칭 신호 감지 로직 (계속)**

**4. 전환 행동 부족 경보**
```
if (전환률_중요_행동의_실행빈도 < 2) → 경보
```

**5. 병원 관심도 급하락 경보**
```
if (품질평균 ≤ 30 || (현재평균 / 이전평균) < 0.7) → 경보
```

**6. 사원별 취약 행동 경보**
```
if (특정행동품질 / 전체행동품질평균 < 0.5) → 경보
```

## 💡 슬라이드 15: Next Best Action

**Next Best Action: 최적 행동 추천**

```
Next Best Action = MAX(행동ᵢ × (성과기여도ᵢ × 긴급도ᵢ × 실행용이성ᵢ))
```

**결정 요소:**
- **성과기여도**: 상관관계 분석 결과
- **긴급도**: 코칭 신호 우선순위
- **실행용이성**: 행동 난이도와 시간 소요

**예시 추천:**
- "지금 이 병원에서는 '질문/탐색' 행동이 가장 효과적입니다"
- "전환률을 높이기 위해 '후속관리' 빈도를 늘려보세요"

# 10_rep_branch_generation_plan.md

목적:
- 가상 `지점`/`담당자` 데이터를 현실적으로 생성하기 위한 실행 계획을 정의한다.
- 목표 수량:
- 전국 지점 10개
- 의원담당자 60명
- 종합병원담당자 45명

기준:
- `AGENTS.md` (territory_code 기준, 추적/정산 플로우, validation-first)
- `08_rebuild_requirements.md` (독립 실행형 파이프라인 요구사항)
- `05_synthetic_data_spec.md` (합성 데이터 재현성 원칙)

적용 단계 정책:
- 1단계(MVP): 중첩구간(overlap) 데이터 생성 금지
- 2단계: overlap 케이스 유도(share 재배분 검증 목적)
- 3단계: 로컬 검토 절차 또는 승인 파일 기준으로 수동 조정/확정 운영

## 4) 현실성 규칙 (Synthetic Realism)

### 4.1 역할별 담당기관 볼륨
- 의원담당자(`rep_role=clinic`):
- 담당 의원 수: 25~60개/인
- 문전약국 연결 수: 35~120개/인

- 종합병원담당자(`rep_role=hospital`):
- 담당 종합병원 수: 2~8개/인
- 연결 문전약국 수: 20~70개/인

### 4.2 경력/생산성 분포
- `hire_date` 분포:
- 1년 미만 15%
- 1~3년 35%
- 3~7년 35%
- 7년 이상 15%

- 생산성 계수(`rep_productivity_factor`, 내부 파생):
- 로그정규 분포 사용(중앙값 1.0, 상하위 편차 허용)
- 의원담당자는 분산 크게, 종합병원담당자는 분산 작게

### 4.3 중첩구간(Share 이슈) 유도
- 동일 `year_quarter x territory_code x brand`에서:
- 종합병원 담당자 1명 + 의원담당자 2~5명 참여 케이스를 20~30% 생성
- 룰 누락 케이스 8~12%를 생성해 `extended` 경로 검증

단계별 적용:
- 1단계(MVP): 위 중첩구간 유도 로직 비활성화(비율 0%)
- 2단계: 위 중첩구간 유도 로직 활성화(비율 20~30%)

## 5) 생성 알고리즘 (실행 순서)

1. 지점 10개 생성 (`dim_branch`)
2. 지점별 headcount 규칙으로 `dim_rep` 생성
3. `dim_rep_assign` 생성 (`rep_id x territory_code x valid_from`)
4. 병원/의원 원천에서 role별 후보 기관 샘플링
5. 담당자-기관 연결 생성(역할별 볼륨 규칙 적용)
6. (2단계부터) 중첩구간 share 케이스 주입
7. run 메타(`run_id`, `seed`, `generated_at`) 분리 저장

재현성:
- `seed` 고정 시 담당자/지점 본문 동일
- `run_id`, `generated_at`은 메타 파일로 분리

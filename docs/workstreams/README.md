# Workstreams Hub

작성일: 2026-03-23

## 목적

`docs/workstreams`는 앞으로의 단계별 작업 메모, 초안, 세부 진행 문서를 모으는 폴더다.

이 폴더를 만든 이유는 간단하다.

- `docs/architecture`는 현재 기준 문서로 유지해야 한다.
- `docs/part1`, `docs/part2`는 이미 과거 단계 보관 폴더로 굳어졌다.
- 따라서 Part3부터는 `docs/part3`, `docs/part4`처럼 단계별 폴더를 계속 늘리지 않는다.

## 사용 원칙

1. 현재 기준 문서는 항상 `docs/architecture`에 둔다.
2. 짧은 작업 안내와 현재 단계 요약은 `docs/ai`에 둔다.
3. 실제 운영 절차는 `docs/runbook`에 둔다.
4. 단계별 작업 메모/초안/세부 계획은 `docs/workstreams` 아래에 둔다.

## 권장 예시

- `docs/workstreams/part3_start_here.md`
- `docs/workstreams/part3_execution_notes.md`
- `docs/workstreams/part3_open_items.md`
- `docs/workstreams/part3_builder_followups.md`


## 금지 원칙

- `docs/part3/`
- `docs/part4/`
- `docs/phase3/`

위와 같은 새 단계 폴더는 만들지 않는다.

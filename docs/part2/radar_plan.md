---

## 🎯 현재 상황 분석

```
Sandbox (월간):
✅ "우리가 어디 있나?" 분석
- CRM + 매출 + 목표 통합
- 다차원 분석 (지점/담당자/품목)
- 월/분기/연간 비교
- 상관관계 분석
→ "분석" 중심

RADAR (월간):
❓ "뭐가 문제인가?" 경보
- 위험 신호 탐지
- 액션 포인트 제시
- 의사결정 지원
→ "의사결정" 중심

문제:
두 개 모두 월간 보고서인데...
차이가 명확하지 않다
```

### **RADAR의 역할 (재정의)**

```
"뭐가 문제인가?" 경보 및 의사결정 지원

구성:
┌──────────────────────────────────┐

│ (경보 + 액션 중심)                │

│    🔴 Critical: 본부 목표 85% 미만 │

│    🟢 Normal: 본부 목표 95% 이상   │
│                                   │
│    현재: 🟡 Warning               │
│    → 목표 대비 93% 달성           │
│    → 선월 대비: -2%               │
│                                   │
│ 2. 근본 원인 (간단명료)           │
│    "목표 미달의 원인이 뭔가?"     │
│                                   │
│    지점별 분석:                   │
│    - 남해영업소: 🟡 (93%)        │
│      → RTR 저하 (68%)            │
│      → 근본: 신규담당자 적응 기간│
│                                   │
│    - 부산영업소: 🟢 (107%)       │
│      → 신약 도입 효과             │
│                                   │
│    - 대구영업소: 🟡 (89%)        │
│      → PV 급감 (경쟁사 공격)     │
│      → 근본: 신제품 런칭 지연    │
│                                   │
│ 3. 즉시 액션 (명확한 지시)        │
│    "뭘 할 것인가?"                │
│                                   │
│    우선순위:                      │
│    1️⃣ 대구영업소 신제품 출시     │
│       (담당: 제약부, deadline: 금주)
│                                   │
│    2️⃣ 남해영업소 신규담당자 코칭 │
│       (담당: 팀장, deadline: 월주)
│                                   │
│    3️⃣ 부산영업소 프로모션 확대   │
│       (담당: 마케팅, deadline: 다음달)
│                                   │
│ 4. 의사결정 옵션 (선택지 제시)    │
│    "우리가 선택할 수 있는 건?"    │
│                                   │
│    Option A: 공격적 (투자 증가)   │
│    → 신규담당자 3명 추가 투입    │
│    → 예상 효과: +2억원            │
│    → 비용: 월 5천만원             │
│                                   │
│    Option B: 수수방관 (현상 유지) │
│    → 다음달까지 자연 회복 기대    │
│    → 리스크: -2억원 추가 손실     │
│                                   │
│    Option C: 선별적 (전략 조정)   │
│    → 대구에만 집중 투자            │
│    → 예상 효과: +1억원            │
│    → 비용: 월 3천만원             │
│                                   │
│    추천: Option A (본부장이 선택) │
│                                   │
│ 특징:                             │
│ ✅ "의사결정 지원"                │
│ ✅ "명확한 액션"                  │
│ ✅ "선택지 제시"                  │
│ ✅ "위험 신호에 빠른 대응"       │
│                                   │
│ 사용자:                           │
│ → 본부장 (의사결정자)             │
│ → 임원진 (경영진)                 │
│                                   │
│ 목적:                             │
│ → "빠른 의사결정"                 │
│ → "선제적 대응"                   │
│ → "위험 최소화"                   │
│                                   │
└──────────────────────────────────┘
```

## 🎯 RADAR의 핵심 차별화

```
Sandbox (분석 보고서):
"우리가 어디 있나요?"
→ 분석가, 기획자가 읽는다
→ 깊이 있고 상세하다
→ 그래프, 표, 필터 많음
→ 원인을 "이해"하는 목적

RADAR (의사결정 보고서):
"우리가 뭘 해야 하나요?"
→ 본부장, 경영진이 읽는다
→ 간결하고 명확하다
→ 경보, 액션, 선택지만 있음
→ 의사결정을 "추진"하는 목적

= 완전히 다른 문서
```

## 📊 포트폴리오 설명 수정

**Slide 09 (수정):**

```
기존:
"RADAR: 위험을 3단계로 본다"

수정:
"RADAR: 의사결정을 위한 경보 시스템"

상세:
┌─────────────────────────────────┐
│ Sandbox vs RADAR               │
│                                 │
│ Sandbox (분석):                │
│ • 지점/담당자/품목 다차원 분석 │
│ • 상관관계 분석                │
│ • 심층 인사이트                │
│ • 사용자: 분석팀               │
│                                 │
│ RADAR (의사결정):              │
│ • 위험 신호 조기 감지          │
│ • 근본 원인 (간단히)           │
│ • 액션 아이템 (명확히)         │
│ • 의사결정 옵션 (선택지)       │
│ • 사용자: 본부장/경영진        │
│                                 │
│ = Sandbox는 "왜?"를 묻고       │
│   RADAR는 "그러면?"로 답한다   │
└─────────────────────────────────┘
```

## 💡 이렇게 차별화하면

```
Sandbox:
"3시간 동안 분석 데이터 분석하기"
→ 분석팀이 사용

RADAR:
"3분 안에 의사결정하기"
→ 본부장이 사용

= 명확한 역할 분리
= 각자 필요한 것만 제공
= 겹치지 않음
```

## 🚀 최종 정리

```
Sandbox:
목적: 분석 (What/Why)
사용자: 분석팀, 기획팀
시간: 월 1회, 상세 분석
형식: 다양한 차트, 필터, 드릴다운

RADAR:
목적: 의사결정 (So What/Now What)
사용자: 본부장, 경영진
시간: 월 1회, 빠른 판단
형식: 경보, 원인, 액션, 선택지

= 완벽하게 분리되어 있다
```

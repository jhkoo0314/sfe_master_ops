# Company Selection UI Plan

작성일: 2026-03-19  
상태: `draft`

이 문서는 Sales Data OS 운영 콘솔에서
`회사 코드 자유 입력` 구조를 `회사 선택 / 회사 등록` 구조로 바꾸기 위한 UI 설계 문서다.

---

## 1. 목적

현재 사이드바는 아래 입력을 직접 받는다.

- 회사 코드
- 회사 이름

이 방식은 개발/검증에는 빠르지만 운영에는 취약하다.

- 오타가 바로 경로 오류가 된다.
- 회사 이름과 회사 코드가 섞인다.
- 같은 회사를 다른 표기로 입력할 수 있다.

따라서 앞으로는:

1. 등록된 회사 목록에서 선택
2. 없으면 신규 회사 등록
3. 내부에서는 고정 `company_key`만 사용

구조로 바꾼다.

---

## 2. 기본 원칙

### 2-1. 사용자는 회사 이름 중심으로 본다

화면에서 기본으로 보여줄 값:
- 회사 이름

보조로 보여줄 값:
- company_key

### 2-2. 시스템은 company_key 중심으로 동작한다

실행/저장/경로는 계속 `company_key` 기준이다.

예:
- `data/company_source/{company_key}/`
- `data/ops_standard/{company_key}/`
- `data/ops_validation/{company_key}/`

### 2-3. 자유 입력은 점진적으로 제거한다

초기에는 호환을 위해 fallback를 남길 수 있지만,
최종 목표는 등록된 회사 선택만 허용하는 구조다.

---

## 3. 화면 구조

현재 사이드바:
- 실행 모드 선택
- 회사 코드 입력
- 회사 이름 입력

변경 후 사이드바 권장 구조:

1. 실행 모드 선택
2. 회사 선택 드롭다운
3. 선택 회사 요약 카드
4. 신규 회사 등록 버튼

---

## 4. 회사 선택 UI

### 표시 형식

드롭다운 라벨 예시:

- `다온파마 (daon_pharma)`
- `한결제약 (hangyeol_pharma)`

### 선택 후 표시 정보

선택 회사 카드:
- 회사 이름
- company_key
- 상태(active/inactive)
- 최근 실행 시각(있으면)

---

## 5. 신규 회사 등록 UI

### 최소 입력

- 회사 이름

### 시스템 처리

1. 정규화 이름 생성
2. 중복 회사 확인
3. 새 `company_key` 발급
4. `company_registry` 저장
5. 바로 현재 선택 회사로 반영

### 등록 완료 후 표시

- `회사 등록 완료`
- `발급된 company_key`

주의:
- 사용자가 직접 company_key를 손으로 정하는 구조는 기본값으로 두지 않는다.

---

## 6. 현재 코드 변경 포인트

영향 파일:

- `ui/console_sidebar.py`
- `ui/console_paths.py`
- `ui/console_runner.py`
- `ui/console_tabs.py`
- `common/company_registry.py` (신규 권장)

---

## 7. 단계별 구현

### Step A. 조회 전용 전환

목표:
- registry에서 회사 목록 조회
- 사이드바 드롭다운으로 선택

이 단계에서는:
- 기존 자유 입력은 임시 fallback로 유지 가능

### Step B. 신규 등록 추가

목표:
- 신규 회사 등록 모달/expander 추가
- 등록 완료 후 즉시 선택 반영

### Step C. 실행 연결

목표:
- 파이프라인/Agent/Builder가 선택된 registry `company_key`만 사용

### Step D. 자유 입력 제거

목표:
- 운영 모드에서는 회사 코드 직접 입력 제거

---

## 8. 예외 처리

### 등록 중복

같은 이름 또는 정규화 이름이 이미 있으면:
- 새 회사 생성 금지
- 기존 회사 선택 유도

### 비활성 회사

`inactive` 회사는:
- 기본 목록에서는 숨기거나
- 선택은 가능하지만 실행은 제한

### 등록 실패

DB 저장 실패 시:
- 명확한 오류 메시지 표시
- 현재 선택 상태 유지

---

## 9. 완료 기준

- 사이드바에서 등록된 회사 목록을 선택할 수 있다.
- 신규 회사를 등록할 수 있다.
- 선택된 회사의 `company_key`가 내부 실행 기준으로 사용된다.
- 회사 이름 오타가 바로 경로 오류로 이어지지 않는다.
- Agent / Pipeline / Builder가 모두 같은 `company_key`를 사용한다.

---

## 10. 한 줄 결론

운영 콘솔의 회사 선택은 더 이상 자유 입력 문자열에 의존하면 안 된다.  
`등록된 회사 선택 + 신규 회사 등록 + 내부 고정 company_key 사용` 구조로 바꿔야 한다.
